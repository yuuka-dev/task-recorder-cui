# 英語対応 (i18n) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** task-recorder-cui の全 UI メッセージを日本語/英語で切替可能にし、ユーザが `--lang en` フラグまたは `config.toml` / `LANG` 環境変数で言語を選べるようにする。

**Architecture:** `src/task_recorder_cui/locales/ja.py` と `en.py` にフラット定数で全メッセージを定義、`i18n.py` の `t(key, **fmt)` ヘルパで参照する。言語決定は `--lang` > `config.ui.lang` > `LC_ALL`/`LANG` > `ja` の優先順位。既存の print 系呼び出しを `t()` ベースに段階的に置換。

**Tech Stack:** Python 3.11+ (tomllib) / 標準ライブラリのみ

**Related Spec:** `docs/superpowers/specs/2026-04-15-i18n-design.md`

**並列開発の注意:** 本プランは timer プラン (`2026-04-15-timer.md`) と並行で進む。`config.py` は timer 側で先行実装されている前提 (依存)。`locales/*.py` にタイマー固有メッセージを追加する作業は timer 側のプランで実施するので、**本プランは既存メッセージの i18n 化に集中** する。

```python
"""i18n ヘルパ。

`t(key, **fmt)` が唯一の公開 API。言語選択は優先順位:
  1. set_lang(...) での明示指定 (CLI フラグ経由)
  2. config.ui.lang
  3. LC_ALL / LANG 環境変数
  4. 'ja' (デフォルト)
未対応言語 (fr など) はサイレントに 'ja' フォールバック。
"""

import os
from typing import Any

from task_recorder_cui.locales import en, ja

_LOCALES: dict[str, object] = {"ja": ja, "en": en}
_current_lang: str | None = None


def set_lang(lang: str | None) -> None:
    """明示的な言語指定。None に戻すと自動判定に戻る。"""
    global _current_lang
    if lang is not None and lang not in _LOCALES:
        raise ValueError(f"unsupported lang: {lang}")
    _current_lang = lang


def current_lang() -> str:
    """現在の有効な言語 ('ja' or 'en') を返す。"""
    if _current_lang is not None:
        return _current_lang
    cfg_lang = _config_lang()
    if cfg_lang in _LOCALES:
        return cfg_lang
    for env_var in ("LC_ALL", "LANG"):
        env = os.environ.get(env_var, "")
        prefix = env.split("_", 1)[0].lower()
        if prefix in _LOCALES:
            return prefix
    return "ja"


def _config_lang() -> str | None:
    """設定ファイルから言語を読む (循環 import 回避のため遅延 import)。"""
    try:
        from task_recorder_cui.config import load_config
    except ImportError:
        return None
    try:
        return load_config().ui.lang
    except Exception:
        return None


def t(key: str, **fmt: Any) -> str:
    """メッセージを現在言語で解決し、{placeholder} を埋める。

    言語モジュールに該当キーが無ければ ja にフォールバック、ja にも無ければ
    key 自体を返す (開発時のタイポを可視化)。
    """
    lang = current_lang()
    mod = _LOCALES.get(lang, ja)
    template = getattr(mod, key, None)
    if template is None and mod is not ja:
        template = getattr(ja, key, None)
    if template is None:
        return key
    return template.format(**fmt)
```

"""locales/ja.py と locales/en.py のキー集合が一致することを保証する。

片方に追加し忘れると CI で落ちる。
"""

import inspect

from task_recorder_cui.locales import en, ja


def _public_string_constants(module) -> set[str]:
    """モジュールから大文字スタートの str 定数名を抽出する。"""
    return {
        name
        for name, value in inspect.getmembers(module)
        if name[0].isupper() and isinstance(value, str)
    }


def test_ja_en_keys_match() -> None:
    ja_keys = _public_string_constants(ja)
    en_keys = _public_string_constants(en)
    missing_in_en = ja_keys - en_keys
    missing_in_ja = en_keys - ja_keys
    assert not missing_in_en, f"en に無いキー: {sorted(missing_in_en)}"
    assert not missing_in_ja, f"ja に無いキー: {sorted(missing_in_ja)}"


def test_all_keys_have_nonempty_values() -> None:
    for module in (ja, en):
        for name in _public_string_constants(module):
            value = getattr(module, name)
            assert value, f"{module.__name__}.{name} が空文字"
```

# task-recorder-cui

A CUI tool to record how you spend your time and visualize it as weekly/monthly averages.
Focused on time tracking, not task management.

[![CI](https://github.com/yuuka-dev/task-recorder-cui/actions/workflows/ci.yml/badge.svg)](https://github.com/yuuka-dev/task-recorder-cui/actions/workflows/ci.yml)
[![PyPI](https://img.shields.io/pypi/v/task-recorder-cui.svg)](https://pypi.org/project/task-recorder-cui/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## Concept

- Record *what / how long / when* in a rough-and-ready way
- Start with 3 categories (game/study/dev); add or remove as you like
- View weekly and monthly averages as plain numbers
- For people who couldn't stick with tools like Toggl — the stripped-down version
- Lightweight enough to use every day without friction

## Main Features

- `tsk start dev "feature X"` to begin a session
- `tsk stop` to end it
- `tsk now`, `tsk today`, `tsk week`, `tsk month`, `tsk range --from ... --to ...`, `tsk all`
- `tsk cat add/remove/restore/rename` for category management
- `tsk --lang en` or `LANG=en_US.UTF-8` for English output
- **Timer**: `tsk start dev "..." --timer 2h30m` — rings a sound after 2h30m
- Interactive menu via `tsk` (no arguments)

## Getting Started

### Requirements
- Python 3.11+
- WSL2 (for the timer feature — Windows speaker output via `powershell.exe`)

### Setup
```bash
pip install task-recorder-cui
```

or from source:
```bash
git clone https://github.com/yuuka-dev/task-recorder-cui
cd task-recorder-cui
pip install -e ".[dev]"
```

### Language

Choose Japanese (default) or English:

```bash
tsk --lang en now
tsk --lang ja now

# or persist via config
tsk config set ui.lang en
```

Priority: `--lang` > `ui.lang` in config > `LC_ALL`/`LANG` env > `ja`.

### Timer (WSL2 only)

```bash
tsk start dev "feature X" --timer 1h
tsk timer set 30m
tsk timer cancel
tsk config set timer.sound_path '/mnt/c/Windows/Media/Alarm01.wav'
tsk config set ui.bar_style rainbow
```

## Limitations

- Timer requires WSL2 with `powershell.exe` and `wslpath` available.
- Timer daemon is a detached child process that survives `tsk` exiting, but
  dies when WSL is fully shut down. Cross-reboot persistence (Windows Task
  Scheduler integration) is planned for Phase 2.2.
